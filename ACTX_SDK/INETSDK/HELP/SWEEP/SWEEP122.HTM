ActiveX Conferencing COM Objects and Control


This document describes the usage models for the COM objects and the ActiveX Control.

COM Objects

There are at least two classes of applications that can use the COM objects:

  • Productivity applications—developers can enable COM-based productivity applications, such as a word processor or a spreadsheet, for conferencing. For example, a word processor could contain a user interface (UI), such as a toolbar button, that starts a conference, then provide another UI that allows document owners to share or transfer their document to others on the conference.
  • Conferencing applications—the COM objects allow a developer to create a full-featured COM-based conferencing application that can start conferences, share applications, transfer files, and so on. The developer creates the UI and harnesses the Microsoft conferencing platform's capabilities underneath.

ActiveX Control

The ActiveX Control can be used by:

  • Visual Basic applications—the control can be used to create a Visual Basic conferencing application that can start conferences, share applications, transfer files, and so on. The control can also be used to enable other Visual Basic applications for conferencing.
  • Scripts (VBScript or JScript) within HTML pages—the key Web-related functionality enabled by the ActiveX Conferencing controls is using a Web page to bring together, in a conference, a community of users with a similar interest.
